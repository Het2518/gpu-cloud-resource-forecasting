# Cloud Project Structure

This workspace is organized for an end-to-end machine learning workflow.

- `data/raw/` stores the source CSVs and should be treated as read-only.
- `data/interim/` is for intermediate feature or parquet artifacts.
- `data/processed/` is for final train/validation/test cuts.
- `notebooks/` contains the numbered analysis and modeling notebooks.
- `outputs/models/` stores exported model weights and metadata.
- `outputs/reports/` stores plots and thesis-ready figures.
- `src_backup/` keeps the transitional Python modules isolated from the notebooks.
